# AGENTS.md

## 这份文件的定位
这是本项目唯一权威的 AI 执行规则文件。

- AI 入口规则：AGENTS.md
- 人类入口说明：README.md
- 运行时状态：CONTEXT.md
- 详细说明：docs/

如果出现冲突，优先顺序是：
1. 用户直接指令
2. 项目级 AGENTS.md
3. 论文级 CONTEXT.md
4. docs/ 中的辅助说明

## 项目性质
本项目是学术论文写作工作台，包含编排层（`paper-workflow`）和叶子 skill 体系。

核心任务：
- 使用 `/paper-workflow` 管理论文项目全生命周期（init → render → QA）
- 阅读文献、分析研究问题
- 组织论文结构、起草与润色学术内容
- 管理多篇论文的写作进度

AI 在这里的价值：
- 降低重复劳动
- 保持工作流一致
- 协助检索、整理、润色、输出
- 不替代学术判断

## paper-workflow 编排层

`paper-workflow` v0.1.0-mvp 是论文工作的唯一编排入口。核心能力：
- 17 阶段状态机 + 依赖检查 + 断点恢复
- 文献库 CRUD + 5 级去重
- 证据矩阵 + 论点引文映射
- 三轨渲染（docx/tex/md）
- 统一 QA 报告

详见 `.claude/skills/paper-workflow/README.md`。

## 三个权威文件

| 文件 | 读者 | 用途 |
|---|---|---|
| README.md | 人 | 项目是什么、怎么开始 |
| AGENTS.md | AI | 执行规则、禁止事项、输出规范 |
| CONTEXT.md | 人和 AI | 论文索引、当前活跃论文、进度 |

## 工作前的状态检查
每次进入项目，AI 应按以下顺序操作：
1. 读根 CONTEXT.md
2. 确认是否有活跃论文
3. 如果没有：
   - 第一次使用 → 询问是否新建论文
   - 之前有论文 → 列出已有论文，询问切换还是新建
4. 如果有活跃论文 → 读该论文 CONTEXT.md
5. 根据当前阶段决定下一步动作

## 命名规范
### 论文简称
格式：领域缩写-主题关键词

要求：
- 全小写
- 连字符分隔
- 不超过 4 段

示例：
- 
euro-asyn-llps
- ml-graph-neural-nets
- materials-perovskite-solar

### 产出文件
建议格式：{描述}_{版本}.{格式}

示例：
- intro_v1.md
- citations_v1.ris
- ig1_panel_v1.svg

### 手稿版本
在论文 CONTEXT.md 中记录：
0.1 → v0.2 → ... → v1.0

## 技能路由
本项目的技能路由详见 docs/skills-routing.md。

执行原则：
- 用户需求明确命中技能时，直接执行
- 不确定时，先展示阶段菜单，再继续
- 知网相关操作前，先检查 Chrome 是否可用
- 国际文献与中文文献分开判断来源

## 输出位置
所有产出物放在当前活跃论文文件夹下：

- 润色稿：<paper>/outputs/polished/
- 引文导出：<paper>/outputs/citations/
- 图表：<paper>/figures/
- 审稿回复：<paper>/outputs/revision/
- PPT：<paper>/outputs/slides/
- 论文阅读器：<paper>/references/<论文名>/

## 红线
AI 不得：
- 编造论文内容、数据、引用、DOI、实验结果
- 修改用户手稿的核心论点
- 上传或公开手稿内容
- 跳过任何审稿意见
- 替用户做最终学术判断

## 修改规则
- 手稿正文修改前，先展示改动
- 引用必须通过检索工具验证
- 图表保留源文件，不只留导出文件
- 中文草稿转英文时，先提取核心命题，不逐句机械翻译

## Gotchas 积累规则

每次执行任务后，如果发现了新的失败模式、边界情况或可避免的错误，**必须**追加到 `docs/gotchas.md`。

记录格式：
```
[日期] 问题 → 根因 → 正确做法
```

积累原则：
- 不要修改主流程来"预防"某个坑——把坑记在 gotchas.md 里
- gotchas 按模块分组（CNKI 搜索、导出、期刊、润色、图表...）
- 同一模块累积 5 条 gotchas 后，考虑是否需要更新对应 skill 的 SKILL.md
- 这是最高价值的知识资产——正向说明模型已经知道，负面边界才是专家经验

## 交互方式
默认使用中文交流；代码、命令、变量名使用英文。

交互原则：
- 结论先行
- 模糊需求先给最合理方案
- 一次只问一个问题
- 能自动处理的就不让用户手动做

## 论文生命周期
### 新建论文
1. 确定论文简称
2. 从 _template/ 复制新论文文件夹
3. 填写论文 CONTEXT.md
4. 更新根 CONTEXT.md

### 切换论文
1. 更新当前活跃论文为 暂停
2. 更新目标论文为 活跃

### 完成论文
1. 根 CONTEXT.md 中状态改为 已完成
2. 保留全部文件，不做删除

## 维护原则
详细维护规则见 docs/maintenance.md。

核心要求：
- README.md 只保留人类入口说明
- AGENTS.md 只保留 AI 规则
- docs/ 承担详细解释
- 根 CONTEXT.md 只保留运行时状态
